# Рекомендованные библиотеки

Выбор инструментов для Python-проектов, ориентированных на производительность, асинхронность
и сопровождаемость. Каждое решение сопровождается обоснованием.

## Веб-фреймворк: FastAPI

- Построен на Starlette и Pydantic.
- Первоклассная поддержка асинхронности.
- Автоматическая генерация OpenAPI-документации.
- Встроенная система DI через `Depends`.

## Валидация и сериализация: Pydantic v2

- Быстрая (на Rust-ядре), строгая типизация.
- Используется для конфигурации, DTO, схем запросов/ответов.
- `pydantic-settings` для загрузки конфигурации из `.env`, YAML.

## ORM и миграции: SQLAlchemy 2.0 + Alembic

- Асинхронный движок (`create_async_engine`).
- Поддержка нативных типов Python, аннотации.
- Alembic для контроля версий БД.
- Репозитории работают с SQLAlchemy Core/ORM, но возвращают доменные модели.

## HTTP-клиент: httpx

- Асинхронный API, поддержка HTTP/2.
- Стандартный интерфейс, близкий к `requests`.

## Работа с БД (альтернативы/дополнения)

- `asyncpg` – прямой драйвер PostgreSQL, если ORM не нужен.
- `redis.asyncio` – для кэширования и сессий.
- `testcontainers` – автоматический запуск контейнеров в интеграционных тестах.

## Фоновые задачи и очереди

- `arq` – простая асинхронная очередь на базе Redis. Предпочтительна для новых проектов.
- `Celery` – если требуется широкая экосистема, мониторинг, но добавляет сложность.

## Логирование

- `loguru` – быстрая настройка, структурированные логи, асинхронная безопасность.
- Всегда выводить JSON в production для удобства парсинга.

## Тестирование

- `pytest` – основной фреймворк.
- `pytest-asyncio` – поддержка асинхронных тестов.
- `pytest-cov` – измерение покрытия.
- `factory_boy` – генерация тестовых данных.
- `Faker` – случайные реалистичные данные.

## Конфигурация

- `pydantic-settings` – чтение переменных окружения, `.env`, поддержка YAML/TOML.
- Конфигурация собирается в одном классе-наследнике `BaseSettings`.

## Документирование кода

- Встроенный OpenAPI FastAPI – для API.
- `mkdocs-material` или `Docusaurus` – для внешней документации (опционально).

## Дополнительно

- `aiofiles` – асинхронная работа с файловой системой.
- `tenacity` – повторные попытки с экспоненциальной задержкой для ненадёжных вызовов.
- `structlog` – альтернатива loguru для продвинутого структурированного логирования.

## Запрещённые или нежелательные библиотеки

- `requests` – только синхронный, используйте `httpx`.
- `psycopg2` – синхронный, замена на `asyncpg` или SQLAlchemy async.
- `Flask` – не поддерживает асинхронность из коробки, предпочтителен FastAPI.
